Overview

ToothFix is the first-ever smartphone application in history that analyzes the PNG version of dental X-Rays and detects dental diseases.

ToothFix is the first-ever smartphone application in history that analyzes the PNG version of dental X-Rays and detects dental diseases. Through cutting-edge technologies such as deep learning and image-processing, ToothFix identifies dental cavities, quality of dental restorations, and periodontal diseases. This ground-breaking smartphone software aims to treat every person on the earth who has a dental disease because tooth decay is inevitable and dental diseases have no age constraints, unfortunately, those numbers are in the billions.

Toothfix provides the following services to users:

  • Detect Tooth Structure.
  • Detect Dental Diseases.
  • Detect the severity of the diseases.
  • Provides recommendations based on the severity.
  • Maintain Dental History.

Research Gap and Research Problems

Dental diseases are one of the quite prominent health-related issues that affect people globally. Recent statistics show that 3.5 Billion people in the global population are affected with dental diseases.

Dental caries (cavities) and severe periodontal disease (gum disease) are a few of the most severe oral diseases that affect people.

Dental health-related disorders have proliferated worldwide due to the excessive intake of fast food and sugary foods, which was followed by bad oral hygiene practices. This is mainly due to poor oral health knowledge of the ordinary people.

Dental diseases can be identified by a dentist using a dental x-ray.

Also, the cost of dental examinations may change based on how critical the condition is, regardless of whether they are not regular. A visit may not require proper treatment and may only require a simple diagnosis but even such a visit can be quite expensive sometimes.

After some careful research, it was discovered that there is no mobile application in the current market that specifically use dental x-rays to give people an overview of their dental health.

Research Objectives

Main objective is to : Create a mobile application to detect diseases related to Stomatology using IntraOral Periapical radiography images to minimize the oral health related problems among people.

Sub Objective 1

Identify the tooth structure and its features (Ex: Average tooth height) using radiography images.

Sub Objective 2

Detect the risk of periodontitis by building a Mask R-CNN model.

Sub Objective 3

Build a Mask R-CNN model to detect the existing dental caries and to identify the possibilities of dental caries.

Sub Objective 4

Identify the quality of restoration types by building a Mask R-CNN model and detecting possible risks associated using the built model.

Methodology





The Figure shows the complete system overview of all four components. The Smart Phone Application is called ToothFix. The application stakeholders include dentists and patients. The application is built using React Native. A registered user of the application needs to upload an IOPA x-ray image through the application. Once this is done, the user can simply select to diagnose this x-ray where the uploaded image will be stored in Firebase and sent to the Cloud Service. The choice of cloud technology was Google Cloud Platform. Google Cloud Platform has the API service and App Engine which enable the deployment of Flask APIs. The four models were exposed as Flask APIs and deployed on Google Cloud Platform App Engine. Auto scaling was enabled to handle multiple requests. The image will be sent as a POST request to be processed by the four models. The models will take the image as input and output the prediction results. In case of a dental disease, if an anomaly is detected using the model, the severity level is calculated and shown to the user. The diagnosis of the image will be done under teeth structure identification, restoration, identification and quality detection, cavity identification and severity detection, and periodontitis identification and severity level detection. The results will be stored in the Firebase database for the user. The results will be sent from Firebase back to the mobile application for the user to view in the next step. The user can view these results and take any necessary actions. The predictions of the image will be done in real-time.
